# Tempo QC: MultiQC Aggregated Reports at Three Levels

> **Quick answer:** Tempo generates MultiQC reports at three levels -- per-sample (`outDir/bams/{SAMPLE_ID}/multiqc/`), per-tumor-normal-pair (`outDir/somatic/{idTumor}__{idNormal}/multiqc/`), and per-cohort (`outDir/cohort_level/{cohort}/`). Each level aggregates different QC metrics and tools.

## Overview

MultiQC is a reporting tool that aggregates outputs from multiple QC tools into a single interactive HTML report. Tempo runs MultiQC at three distinct levels, each covering a different scope of analysis. The three levels are complementary and are explicitly cross-referenced within each report's introduction text.

## Level 1: Per-Sample MultiQC (`SampleRunMultiQC`)

**Location:** `outDir/bams/{SAMPLE_ID}/multiqc/`

**What it aggregates:**
- **fastp** -- FASTQ-level quality metrics from all lanes for this sample (JSON files)
- **Alfred** -- BAM alignment statistics (both aggregate and per-read-group TSV files, parsed via `parse_alfred.py`)
- **Qualimap** -- Coverage statistics extracted from `genome_results.txt`
- **CollectHsMetrics** -- On-target hybridization metrics (exome assays only)

**Output files:**
- `multiqc_report.html` -- Interactive HTML report
- `multiqc_data.zip` -- Raw data behind the report
- `{SAMPLE_ID}.QC_Status.txt` -- Pass/warn/fail status for key metrics

This report provides a complete picture of sequencing and alignment quality for a single sample. It does not include any tumor-normal pairing QC (concordance, contamination, or FACETS). The report's introduction text directs users to the somatic-level and cohort-level reports for pairing QC.

## Level 2: Per-Pair Somatic MultiQC (`SomaticRunMultiQC`)

**Location:** `outDir/somatic/{idTumor}__{idNormal}/multiqc/`

**What it aggregates:**
- **Conpair** -- Concordance and contamination metrics for the tumor-normal pair
- **Qualimap** -- Coverage data for both tumor and normal (used to display tumor/normal coverage side by side)
- **FACETS** -- Copy number QC summary and FACETS run quality metrics

**Output files:**
- `multiqc_report.html` -- Interactive HTML report
- `multiqc_data.zip` -- Raw data behind the report (contains: multiqc_conpair.txt, multiqc_facets_qc.txt, multiqc_facets_run_summary.txt, multiqc_qc_criteria.txt, multiqc_general_stats.txt, multiqc_sources.txt, multiqc_data.json)
- `{idTumor}__{idNormal}.QC_Status.txt` -- Pass/warn/fail status

This report focuses exclusively on pairing-level QC. It does not re-include the FASTQ or per-sample alignment metrics. The report directs users to the bam-level reports for FASTQ/alignment QC and to the cohort-level report for the full picture.

The general stats table in this report includes columns for:
- Tumor and Normal sample identifiers
- Tumor and Normal contamination percentages
- Concordance score
- Tumor and Normal coverage

## Level 3: Cohort MultiQC (`CohortRunMultiQC`)

**Location:** `outDir/cohort_level/{cohort}/`

**What it aggregates:**
This is the most comprehensive report, combining metrics from all samples and all pairs in the cohort:
- **fastp** -- FASTQ metrics for all tumor and normal samples
- **Alfred** -- Alignment metrics for all samples (parsed via `parse_alfred.py`)
- **Qualimap** -- Coverage for all samples
- **CollectHsMetrics** -- HS metrics for all samples (exome only)
- **Conpair** -- Concordance and contamination for all tumor-normal pairs
- **FACETS** -- Copy number QC for all pairs

**Output files:**
- `multiqc_report.html` -- Interactive HTML report
- `multiqc_data.zip` -- Raw data behind the report

This report is only generated when the pipeline is run with the `--aggregate` flag. For cohorts with more than 50 samples, it switches from an interactive table to a beeswarm plot to keep the report manageable. The `max_table_rows` parameter is dynamically set based on the number of samples.

## QC Status Files

The sample-level and somatic-level MultiQC processes each produce a `QC_Status.txt` file generated by the `general_stats_parse.py` script. This file contains pass/warn/fail classifications for key metrics based on predefined criteria, providing a quick at-a-glance quality assessment without needing to open the HTML report.

## File Locations

```
# Per-sample
outDir/bams/{SAMPLE_ID}/multiqc/
├── multiqc_report.html
├── multiqc_data.zip
└── {SAMPLE_ID}.QC_Status.txt

# Per-pair
outDir/somatic/{idTumor}__{idNormal}/multiqc/
├── multiqc_report.html
├── multiqc_data.zip
└── {idTumor}__{idNormal}.QC_Status.txt

# Cohort-wide
outDir/cohort_level/{cohort}/
├── multiqc_report.html
└── multiqc_data.zip
```

## Example

```bash
# Open the sample-level report
open outDir/bams/DU874145-T/multiqc/multiqc_report.html

# Check quick QC status for a sample
cat outDir/bams/DU874145-T/multiqc/DU874145-T.QC_Status.txt

# Open the somatic-level report for a pair
open outDir/somatic/DU874145-T__DU874145-N/multiqc/multiqc_report.html

# Open the cohort-level report
open outDir/cohort_level/default_cohort/multiqc_report.html
```
