# Neoantigen Prediction with NetMHCpan in Tempo

> **Quick answer:** Tempo predicts neoantigens for each somatic mutation using NetMHCpan, leveraging the patient-specific HLA alleles determined by Polysolver to estimate which mutant peptides are likely to be presented on MHC class I molecules.

## What Neoantigen Prediction Does

Neoantigens are novel peptides generated by somatic mutations that can be presented on the tumor cell surface via HLA class I molecules, potentially triggering a T-cell immune response. Neoantigen prediction is clinically relevant for immunotherapy patient selection, neoantigen vaccine design, and understanding immune evasion mechanisms.

The `RunNeoantigen` process in Tempo runs a Python-based neoantigen prediction pipeline that internally calls NetMHCpan. NetMHCpan uses artificial neural networks trained on peptide-MHC binding data to predict the binding affinity of mutant peptides to specific HLA alleles.

## Inputs

The process requires two main inputs:

1. **Polysolver HLA file**: The `{idNormal}.hla.txt` file containing the patient's six HLA class I alleles (two each for HLA-A, HLA-B, HLA-C). This ensures predictions are personalized to the patient's immune system.

2. **Somatic MAF file**: The annotated somatic mutation file. Each non-synonymous mutation generates candidate peptides (typically 8-11 amino acids) that are evaluated for HLA binding.

Additionally, the process uses reference cDNA and CDS sequence files (`neoantigenCDNA` and `neoantigenCDS`) to translate mutant codons into peptide sequences.

## How the Process Works

The `neoantigen.py` script orchestrates the prediction workflow:

1. Reads the somatic MAF and extracts non-synonymous mutations
2. Generates mutant peptide sequences of varying lengths
3. Runs NetMHCpan to predict binding affinity for each peptide against all six patient HLA alleles
4. Compiles results into a single predictions file

The process is multi-threaded (`${task.cpus}`) and includes special handling for large MAF files on the Juno HPC, with dynamic wall time allocation based on file size (files over 10 MB receive the maximum wall time).

After NetMHCpan completes, the pipeline prepends a `sample` column containing the tumor-normal pair identifier to each row of the predictions file.

## Output File

The primary output is a tab-delimited text file with 13 columns:

| Column | Description |
|--------|-------------|
| `sample` | Tumor-normal pair identifier |
| `algorithm` | Prediction algorithm (e.g., `NetMHCpan-4.0`) |
| `hla_allele` | HLA allele tested (e.g., `HLA-A*32:01`) |
| `peptide` | Mutant peptide sequence |
| `core` | Binding core sequence |
| `icore` | Interaction core sequence |
| `score` | Binding score (0-1, higher = stronger binding) |
| `affinity` | Predicted binding affinity in nM (lower = stronger) |
| `rank` | Percent rank against random peptides |
| `binder_class` | Classification: Strong Binder, Weak Binder, or Non Binder |
| `best_binder_for_icore_group` | Whether this is the best binder for its icore group |
| `is_in_wt_peptidome` | Whether the peptide exists in the wild-type proteome |
| `neo_maf_identifier_key` | Links back to the somatic MAF mutation (format: `TUMOR_chrom_pos-pos_ref_alt`) |

The process also produces a neoantigen-annotated MAF file (`{idTumor}__{idNormal}.neoantigens.maf`) that is used for downstream MAF annotation.

## File Locations

```
outDir/somatic/{idTumor}__{idNormal}/neoantigen/{idTumor}__{idNormal}.all_neoantigen_predictions.txt
```

The neoantigen-annotated MAF is used internally and is not separately published to the neoantigen directory.

## Example

```bash
# The pipeline internally executes:
python /usr/local/bin/neoantigen/neoantigen.py \
  --config_file /usr/local/bin/neoantigen/neoantigen-docker.config \
  --sample_id sample_T__sample_N \
  --hla_file sample_N.hla.txt \
  --maf_file sample_T__sample_N.somatic.final.maf \
  --threads 4 \
  --output_dir neoantigen
```
